# إرشادات المساهمة في Stampcoin Platform

شكراً لمشاركتك في تطوير Stampcoin Platform! هذا المستند يوضح كيف يمكنك المساهمة في المشروع.

## الخطوات الأساسية للمساهمة

1. **أنشئ حساباً على GitHub** إذا لم ي لديك بالفعل.
2. **أنشئ fork** للمستودع.
3. **استنسخ المستودع** إلى جهازك المحلي:
   ```bash
   git clone https://github.com/your-username/stampcoin-platform.git
   cd stampcoin-platform
   ```
4. **أنشئ فرعاً جديداً** لتغييراتك:
   ```bash
   git checkout -b feature/your-feature-name
   ```
5. **قم بتطويرك وتغييراتك**.
6. **أضف تغييراتك** إلى git:
   ```bash
   git add .
   git commit -m "Add your feature description"
   ```
7. **ادفع تغييراتك** إلى فرعك:
   ```bash
   git push origin feature/your-feature-name
   ```
8. **أنشئ طلب سحب (Pull Request)** في GitHub.

## معايير الكود

### أسلوب الكود

- استخدم المسافات بدلاً من علامات التبويب (Tabs).
- استخدم قوالب ESL القياسية.
- احتفظ بالأسطر أقل من 80 حرفاً.
- استخدم camelCase للمتغيرات والدوال.
- استخدم Pascal للفئات.

### التعليقات

- استخدم JSDoc للدوال والفئات.
- اشرح المنطق المعقد في التعليقات.
- تجنب التعليقات التي تشرح ما يفعله الكود بوضوح (يجب أن يكون الكود واضحاً بنفسه).

### اختبار الكود

- أكتب اختبارات لوحدات العمل (unit tests) لجميع الميزات الجديدة.
- حافظ على تغطية الاختبار فوق 80%.
- نفذ الاختبارات قبل كل طلب سحب:
  ```bash
  npm test
  ```

## أنواع المساهمات

### الميزات الجديدة

- ابدأ باسم "feature/" للفرع الجديد.
- قدم وصفاً واضحاً للميزة في طلب السحب.
- قدم أمثلة على الاستخدام.

### إصلاحات الأخطاء

- ابدأ باسم "fix/" للفرع الجديد.
- اشرح المشكلة التي تم حلها في طلب السحب.
- قدم اختبارات للتحقق من إصلاح الخطأ.

### التحسينات

- ابدأ باسم "improvement/" للفرع الجديد.
- اشرح التحسين الذي تم إجراؤه ولماذا هو مفيد.

## مراجعة طلبات السحب

سيتم مراجعة جميع طلبات السحب من قبل أعضاء الفريق. قد يطلبون تعديلات قبل الاندماج. نحن نقدر مساهمتك ونحن هنا للمساعدة!

## التواصل

إذا كان لديك أي أسئلة، يمكنك:
- إنشاء issue في المستودع.
- المشاركة في محادثتنا على Discord.
- إرسال بريد إلكتروني إلى: contributors@stampcoin.com

شكراً لمساهمتك!
